## Version History Format

When adding new versions, use this structure:

### Added
- New features or functionality

### Changed
- Changes in existing functionality

### Deprecated
- Features that will be removed in upcoming releases

### Removed
- Features that have been removed

### Fixed
- Bug fixes

### Security
- Security vulnerability fixes

---

**Note**: When you use the template sync workflow, the VERSION file will automatically be updated in your pull requests, making it clear when a new version is available.
